《图解HTTP》笔记（一）

发表于 2017-03-15

虽然做了很久iOS开发了，但到目前对于网络这一块还是不知其所以然，最近也是下决心补一下这块知识，所以也是选择了《图解HTTP》作为入门书籍。

*****

了解Web及网络基础

![][image-1]

如上图所示，一个Web页面能显示出来需要一个客户端和一个服务器。而通过发送请求获取服务器资源的Web浏览器等，都可以成为客户端（client）。

而我们平时经常接触到的HTTP协议就是作为一个规范标准以完成从客户端到服务器端等的一系列运作流程。可以说Web是建立在HTTP协议上进行通信的。

一个标准的网络请求过程

  1. 客户端输入URL
    一个标准的URL组成部分为
    • 协议（比如HTTP，FTP）
    • 域名（比如baidu.com）
    • 文件路径（比如/HDD/hexo/_post/a.MD）
    • 端口（比如:4000）
  2. 查找域名的IP
    1. 查找本地hosts文件
    2. DNS查询
      1. 询问本地域名服务器
      2. 询问根域名服务器
    3. 返回服务器IP地址
  3. 客户端向服务器的IP地址发送一个HTTP请求
  4. 服务器处理请求
  5. 服务器发回一个响应
  6. 客户端开始处理响应

  7. 客户端开始显示响应

    详情可以看：

    • 当···时发生了什么？
    • 在浏览器地址栏输入一个URL后回车，背后会进行哪些技术步骤？
